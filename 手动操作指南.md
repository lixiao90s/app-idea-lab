## Step 2：创建 Cloudflare API Token

> Token 用于在本地运行 `wrangler` 部署 Worker、创建 KV。  
> **只需创建一次**，后续 Step 3、Step 4 都会用到。

### 2.1 打开 API Token 页面

1. 浏览器登录 [Cloudflare Dashboard](https://dash.cloudflare.com/)
2. 右上角头像 → **My Profile**（我的个人资料）
3. 左侧菜单 → **API Tokens**
4. 或直接打开：https://dash.cloudflare.com/profile/api-tokens

### 2.2 选择创建方式（推荐模板）

**方式 A — 使用官方模板（推荐，最简单）**

1. 点击 **Create Token**
2. 找到模板 **Edit Cloudflare Workers**，点击 **Use template**
3. 页面会自动勾选所需权限，一般无需修改
4. 跳到 **2.4** 继续

**方式 B — 自定义 Token**

1. 点击 **Create Token** → **Create Custom Token**
2. **Token name** 填：`app-idea-lab-deploy`（便于日后识别）
3. 在 **Permissions** 区域点击 **Add more**，添加以下权限：

| 资源类型 | 权限 | 说明 |
|----------|------|------|
| Account | Workers Scripts → **Edit** | 部署 / 更新 Worker |
| Account | Workers KV Storage → **Edit** | 创建 KV、读写缓存 |
| Account | Account Settings → **Read** | wrangler 识别账号（建议加上） |

4. **Account Resources** 选择：**Include** → 你的 Cloudflare 账号（通常只有一个）
5. **Zone Resources**（区域资源）：选 **All zones** 或 **All zones from an account** 均可；部署 Worker **不强制**绑定某个域名 zone，但 wrangler 有时需要读取账号信息，保持默认即可
6. **Client IP Address Filtering**（IP 限制）：留空，不限制
7. **TTL**（有效期）：可留空（永久有效），或设 90 天到期后重新创建

### 2.3 确认权限清单

创建前核对 Token 至少包含：

- [ ] `Workers Scripts:Edit`
- [ ] `Workers KV Storage:Edit`

若使用 **Edit Cloudflare Workers** 模板，以上已自动包含。

### 2.4 生成并复制 Token

1. 滚动到页面底部，点击 **Continue to summary**
2. 检查摘要无误后，点击 **Create Token**
3. **立即复制** 显示的 Token 字符串（只显示一次，关闭后无法再次查看）

> ⚠️ Token 相当于密码，不要写入 Git、不要发到聊天群。丢失只能重新创建。

### 2.5 配置认证（二选一）

> 同一 Cloudflare 账号可部署多个 Worker；不会因为你以前部署过别的站就冲突。  
> 已检查本机 `~/.zshrc` 里没有旧的 `CLOUDFLARE` 配置，`bad assignment` 是 **当前这条命令写法** 的问题，不是历史项目导致的。

#### 方式 B：使用 API Token（2.4 创建的 Token）

**不要从本文档整行复制** `export ...` 命令——Markdown 里的引号可能是弯引号 `‘’`，粘贴到终端会报 **`bad assignment`**。

请 **手敲** 下面步骤（引号用键盘英文键，在 `L` 右边）：

```bash
# 第 1 步：手敲到左引号为止（末尾是英文单引号 '）
export CLOUDFLARE_API_TOKEN='

# 第 2 步：粘贴 Token（仅粘贴 Token 本身，不要带空格/换行）

# 第 3 步：手敲右引号 ' 然后回车
'
```

或用交互输入（同样避免复制引号）：

```bash
read -s 'CLOUDFLARE_API_TOKEN?粘贴 Cloudflare Token 后回车: '
export CLOUDFLARE_API_TOKEN
echo
```

验证：

```bash
cd /Users/lixiao/MyWorkSpace/WebProjs/appstore-idea-find-main/worker
npx wrangler whoami
```

正常示例：

```
👋 You are logged in with an API Token, associated with the email 'your@email.com'!
```

- **方式 A（login）**：已写入 `~/.wrangler/`，关终端也有效，无需额外配置
- **方式 B（Token）**：可写入 `worker/.env`（见 2.5 方式 C），或写入 `~/.zshrc`（须手敲英文引号，勿从文档复制）

### 2.7 常见问题

| 现象 | 处理 |
|------|------|
| `bad assignment` / `xxx not found` | **不是旧网站冲突。** 常见原因：`export VAR =值`（`=` 前有空格）、弯引号 `‘’`、从文档整行复制。正确：`export CLOUDFLARE_API_TOKEN='token'`（**等号两侧无空格**） |
| `Authentication error` | Token 复制不完整（首尾多空格），或已过期，重新创建 |
| `Insufficient permissions` | 补加 **Workers KV Storage Edit** 权限后重建 Token |
| `wrangler: command not found` | 在 `worker/` 目录用 `npx wrangler whoami`，或先 `cd worker && npm install` |
| 找不到 Edit Cloudflare Workers 模板 | 用 **2.2 方式 B** 手动添加权限 |
| 已 login 还要 Token 吗？ | **不需要**。`wrangler login` 成功后可直接 Step 3 |

### 2.8 完成检查

- [ ] 已 **login** 或已正确配置 **Token / .env** 之一
- [ ] 在 `worker/` 目录执行 `npx wrangler whoami` 显示登录成功

## Step 6：Cloudflare Pages + 绑定 idea.lx06.com

### 6.1 创建 Pages 项目

> Cloudflare 新版界面里，**Production branch / Build output directory** 不在首页，而在选完仓库后的 **「Set up builds and deployments」** 页面。往下滚动才能看到。

#### 6.1.1 进入创建流程

1. 打开 [Cloudflare Dashboard](https://dash.cloudflare.com/) → 左侧 **Workers & Pages**
2. 点击 **Create application**（创建应用）
3. 切到 **Pages** 标签页
4. 选 **Import an existing Git repository**（导入已有 Git 仓库）  
   > 旧版文案可能是 **Connect to Git**，是同一个入口
5. 授权 GitHub，选择仓库 **`lixiao90s/app-idea-lab`**
6. 点击 **Begin setup**（开始设置）

#### 6.1.2 填写构建设置（关键页面）

进入 **Set up builds and deployments** 后，向下滚动，填写：

| 字段 | 填什么 | 说明 |
|------|--------|------|
| **Project name** | `app-idea-lab` | 会生成 `app-idea-lab.pages.dev` |
| **Production branch** | `main` | 下拉选 `main`；若下拉为空，说明仓库还没 push 过分支 |
| **Framework preset** | **None** | 纯静态 HTML，不要选 React/Vite 等 |
| **Build command** | `exit 0` | 静态站官方推荐写法（不要留空） |
| **Build output directory** | **留空** | 根目录直接部署；**不要填 `/`** |
| **Root directory (advanced)** | 留空 | 项目在仓库根目录 |

> 若看不到 **Build output directory**：先选 **Framework preset → None**，或展开 **Root directory (advanced)** 区域，字段在页面下方。

#### 6.1.3 部署

1. 点击 **Save and Deploy**
2. 等待构建完成（静态站通常 1 分钟内）
3. 点击 **Continue to project**，获得 `https://app-idea-lab.pages.dev` 临时地址

#### 6.1.4 创建后修改构建设置（若 6.1.2 漏填）

已创建项目的话，进入：

**Pages 项目 → Settings → Builds & deployments → Build configuration → Edit**

同样设置为：`Build command = exit 0`，`Build output directory` 留空，`Production branch = main`。

### 6.2 绑定自定义域名 idea.lx06.com

1. Pages 项目 → **Custom domains** → **Set up a custom domain**
2. 输入：**`idea.lx06.com`**
3. Cloudflare 会自动在 `lx06.com`  zone 添加 DNS 记录（通常是指向 Pages 的 CNAME）
4. 等待 SSL 证书状态变为 **Active**（通常几分钟）

### 6.3 DNS 记录参考（若需手动添加）

在 Cloudflare → **lx06.com** → **DNS** 中确认存在：

| 类型 | 名称 | 目标 | 代理 |
|------|------|------|------|
| CNAME | `idea` | `app-idea-lab.pages.dev`（或 Pages 提示的目标） | 已代理 ☁️ |

> **不要**把 `lx06.com` 根域名指到本工具，游戏官网和工具站分开。

### 6.4 验证

浏览器访问：**https://idea.lx06.com/** 应打开 App Idea Lab 首页。

### 7.1 添加 Search Console 资源（勿选错类型）

1. 打开 [Google Search Console](https://search.google.com/search-console/welcome)
2. 选左侧 **「网址前缀」**（不是「网域」）
3. 输入：**`https://idea.lx06.com`**
4. 点击 **继续**

> ⚠️ **不要选「网域」`lx06.com`**。  
> 网域验证要在根域名 `@` 加 TXT，容易和 pick10 主站 DNS 搞混，且报错「找不到验证令牌」很常见。  
> 本工具站只需验证子域名，用 **网址前缀** 即可。

### 7.2 所有权验证（推荐 HTML 文件，最简单）

Search Console 会给出验证方式，**优先选「HTML 文件」**：

1. 下载 Google 提供的文件，例如 `google1234567890abcdef.html`
2. 放到项目**根目录**（与 `index.html` 同级）：

```
app-idea-lab/
├── index.html
├── google1234567890abcdef.html   ← 放这里
├── robots.txt
└── sitemap.xml
```

3. 提交并 push 到 GitHub，等 Cloudflare Pages 自动部署（约 1～2 分钟）
4. 浏览器访问 `https://idea.lx06.com/google1234567890abcdef.html` 能打开后，回到 Search Console 点 **验证**

### 7.3 备选：DNS 验证（若坚持用 DNS）

仅在 **网址前缀** `https://idea.lx06.com` 下选择「域名提供商」验证时：

1. Cloudflare → **lx06.com** → **DNS** → **Add record**
2. 填写：

| 类型 | 名称 | 内容 | 代理 |
|------|------|------|------|
| TXT | `idea` | Google 给的整段 `google-site-verification=...` | 仅 DNS（TXT 无代理开关） |

3. **不要**填到 `@`（那是根域名，用于「网域」验证）
4. 保存后等 **5～30 分钟** 再点验证（DNS 传播需要时间）

若仍失败，用 `dig` 自查（把验证码换成你的）：

```bash
dig TXT idea.lx06.com +short
```

应能看到 `google-site-verification=...`。

### 7.4 提交 Sitemap

验证成功后：

1. Search Console 左侧 **站点地图**
2. 输入：`https://idea.lx06.com/sitemap.xml`
3. 提交

## Step 9：Google AdSense

### 9.1 申请（详细步骤）

> 变现可选，但建议等 **Step 6 域名上线**、**Step 7 Search Console 验证通过** 后再申请，通过率更高。

#### 9.1.1 申请前检查

确认以下页面可公开访问：

- [ ] https://idea.lx06.com/
- [ ] https://idea.lx06.com/about.html
- [ ] https://idea.lx06.com/privacy.html（AdSense 要求有隐私政策）

填写网站时使用 **`https://idea.lx06.com`**（带 `https://`），**不要**填：

- `idea.lx06.com`（缺协议）
- `xxx.pages.dev`（临时地址）
- `lx06.com`（根域名是游戏官网）

#### 9.1.2 注册 / 登录 AdSense 账号

1. 打开 https://www.google.com/adsense/
2. 用 Google 账号登录（建议与 Search Console 同一账号）
3. 若是**新账号**，按向导完成：
   - 选择国家/地区（与收款地址一致，**创建后难改**）
   - 填写联系信息
   - 关联收款方式（银行卡等，可先填后补，但审核前建议填完）

#### 9.1.3 添加网站

1. 登录后进入 AdSense 首页，或左侧 **网站**（Sites）
2. 点击 **添加网站**（Add site）
3. 输入：**`https://idea.lx06.com`**
4. 点击 **保存并继续**

站点会出现在列表中，状态多为 **正在准备** 或 **需要审核**（Requires review）。

#### 9.1.4 验证网站所有权（三选一）

AdSense 会要求证明你拥有该站，任选一种即可：

---

**方式 A — AdSense 代码（推荐）**

1. 在添加网站流程中选择 **AdSense 代码**（AdSense code snippet）
2. 复制 Google 提供的一段 `<script>`，形如：

```html
<script async src="https://pagead2.googlesyndication.com/pagead/js/adsbygoogle.js?client=ca-pub-xxxxxxxxxxxxxxxx"
     crossorigin="anonymous"></script>
```

3. 粘贴到 **`index.html` 的 `<head>` 内**（`</head>` 之前），例如放在 `</head>` 上一行
4. 同样建议加到 `about.html`、`privacy.html` 的 `<head>`（AdSense 建议多页放置）
5. commit 并 push，等 Cloudflare Pages 部署完成（1～2 分钟）
6. 浏览器打开首页 → 右键 **查看网页源代码**，确认 `<head>` 里能看到该 `<script>`
7. 回到 AdSense，勾选「我已完成」，点击 **验证**（Verify）

> 申请阶段只需验证代码，**先不要**改 `ADS_ENABLED: true`（见 9.2，等审核通过再开）。

---

**方式 B — ads.txt 文件**

1. 选择 **ads.txt 代码**
2. Google 会给一行类似：

```
google.com, pub-xxxxxxxxxxxxxxxx, DIRECT, f08c47fec0942fa0
```

3. 在项目根目录新建 `ads.txt`，粘贴上述内容
4. push 后访问 https://idea.lx06.com/ads.txt 能打开即成功
5. 回 AdSense 点 **验证**

---

**方式 C — Search Console（若 Step 7 已完成）**

1. 选择通过 **Search Console** 验证
2. 确保 Search Console 里 **`https://idea.lx06.com`** 已显示「已验证」
3. 同一 Google 账号登录 AdSense，直接点 **验证**

#### 9.1.5 提交审核

验证通过后：

1. 点击 **请求审核**（Request review）
2. 若提示选择 **CMP**（同意管理平台 / Cookie 合规），按你站点受众选择；纯工具站可先选 Google 提供的默认方案或后续再配
3. 点击 **提交**

#### 9.1.6 等待审核与查看状态

1. AdSense → **网站** → 点击 `idea.lx06.com` 查看状态
2. 常见状态：

| 状态 | 含义 |
|------|------|
| 正在准备 / Getting ready | 还需完成账号资料或站点验证 |
| 需要审核 / Requires review | 已提交，等待 Google 人工审核 |
| 已准备好 / Ready | 审核通过，可展示广告（进入 9.2） |
| 需要注意 / Needs attention | 有政策或技术问题，点进去看邮件和详情 |

## 常见问题

### Q：idea.lx06.com 和 lx06.com 会冲突吗？

不会。根域名继续指向游戏官网，只有 `idea` 子域名指向 Pages。

### Q：需要单独买域名吗？

**不需要**。`lx06.com` 已在 Cloudflare 管理即可。

### Q：AdSense 要填哪个域名？

填 **`idea.lx06.com`**，不要填 `pages.dev` 临时地址。

### Q：Worker 部署报 Token 错误

优先在 `worker/` 目录执行 `npx wrangler login` 重新登录；若用 Token，确保 `worker/.env` 或 `export` 使用英文直引号 `'...'`。

### Q：评论加载失败

确认 `API_PROXY` 已 push，且 `https://WorkerURL/health` 正常。
